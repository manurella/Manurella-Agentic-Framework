## Evidence Assessment

The strongest report supports a conservative Hybrid Workspace Controller: typed state, bounded action, retrieval-first context compilation, external verification, evidence-gated learning, and a single-controller default with temporary specialists.

The more theory-heavy report contains useful candidate mechanisms but overstates emerging evidence. Global Workspace Theory, active inference, ODAR, AgentVisor, TOKI, Roynard persistence semantics, and hardware-specific consciousness arguments are not accepted as production foundations without local replication and cost-adjusted benchmark wins.

## Promoted Conclusions

Promote into the Brain specification:

1. Separate Brain, Framework Atlas, runtime, models, memory stores, tools, domains, and interface.
2. Treat the Brain as an online cognitive controller rather than an LLM, prompt, or permanent society of agents.
3. Maintain typed task, world, user, self, uncertainty, and capability state.
4. Use a transient active workspace with claims, evidence, plans, contradictions, and budgets.
5. Compile small typed context packets rather than accumulating an unbounded transcript.
6. Select cognitive strategy based on the task instead of enforcing one reasoning ideology.
7. Prefer external evidence and deterministic verification over intrinsic self-correction.
8. Control confidence, retries, escalation, stopping, risk, and resources explicitly.
9. Gate all persistent learning by provenance, schema, evidence, permission, and review.
10. Use one controller by default and temporary specialists only for separable, mergeable work.

## Production Architecture

The accepted top-level production hypothesis is:

```text
Hybrid Workspace Controller
|- Interpreter
|- State Estimator
|- Active Workspace And Evidence Graph
|- Context Compiler
|- Cognitive Strategy Selector
|- Execution Controller
|- Verifier And Repair Manager
|- Metacognitive Governor
`- Evidence-Gated Learning Controller
```

This structure is an implementable systems architecture. Cognitive theories may inspire individual policies, but no theory is allowed to become an architectural dependency without evidence.

## Runtime Loop

```text
Intake and policy gate
-> Interpret and frame
-> Estimate state and uncertainty
-> Compile context
-> Select cognitive strategy
-> Plan or act
-> Observe
-> Verify and repair
-> Learn, consolidate, or forget
-> Stop, escalate, ask, or repeat
```

The loop is explicit, bounded, observable, resumable, and testable at every transition.

## Mode And Effort Correction

Both reports proposed Fast policies that reduce research or final depth. Manurella rejects that coupling.

- Fast and Standard are delivery topologies with the same final acceptance contract.
- Fast produces an early usable artifact and then improves it through resumable layers.
- Standard develops through a conventional plan-first sequence before consolidation.
- Low, Medium, High, Max, Ultra, and Sentient control cognitive and resource policy independently from mode.

Fast plus Ultra is valid. Standard plus Low is valid. Safety and required correctness are never optional.

## Private Sentient Branch

Sentient remains the private conscious-architecture research objective. Selecting Sentient enters an isolated research runtime rather than merely raising a token budget.

The research branch may investigate functional correlates, self-modeling, global availability, metacognition, continual learning, perturbation response, competing consciousness theories, and alternative substrates. No one theory, proxy, benchmark, or hardware claim establishes phenomenal consciousness by itself.

The branch requires separate memory, policies, logs, permissions, ethics review, and falsification criteria. It cannot silently share experimental learning with production Manurella.

## Research Candidates

Keep behind experiments:

- literal Global Workspace broadcast implementation
- active inference or Expected Free Energy as a planning policy
- graph-first semantic memory
- exact salience and confidence formulas
- permanent specialist swarms
- AgentVisor-style semantic virtualization
- TOKI contradiction algebra
- autonomous procedural-memory revision
- offline model fine-tuning
- consciousness or moral-patienthood claims
- neuromorphic hardware requirements

## Falsification Rule

Every added cognitive mechanism must outperform a simpler baseline on relevant behavior at acceptable cost, latency, reliability, and risk. If it does not, Manurella removes it from active use regardless of theoretical elegance.

## Next Research Path

Broad Brain research is complete enough to specify the top-level kernel. Future research is subsystem-specific and begins with:

Interpreter -> Task Model -> Acceptance Contract
