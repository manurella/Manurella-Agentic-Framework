# Domain Packs

Domain packs are portable specialist definitions. They are not runtime-specific agent files.

Initial packs:

- `core`
- `build`
- `muse`
- `pixel`
- `mentor`

Each domain pack should eventually define:

- purpose
- when to use
- when not to use
- required context
- permissions
- output contracts
- rubrics
- examples
- runtime adapter notes
